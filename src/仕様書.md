# My SoftBank 料金明細PDF自動ダウンロード 仕様書

最終更新: 2026-03-15

---

## 1. 概要

My SoftBankポータルにPlaywrightで自動ログインし、指定月の請求書PDFをダウンロードしてローカルに保存するPythonスクリプト。

### 対応範囲
- My SoftBankの2段階SMS認証（acr_value=1 / acr_value=2）
- Googleスプレッドシート（サービスアカウント認証）で管理した複数アカウントの一括処理
- 電子帳簿保存法準拠のファイル命名
- アカウントごとに独立したセッション保存による再認証スキップ

## 3. スプレッドシート構成

スプレッドシートへのアクセスはGCPサービスアカウント（`service_account.json`）で認証。
URLは `.env` の `SPREADSHEET_URL` に設定する。

### アカウントシート
| 列 | 内容 |
|---|---|
| 電話番号 | SoftBank ID（ハイフンあり/なしどちらでもOK） |
| パスワード | My SoftBankのパスワード |
| PDFの種類 | `電話番号別` / `一括` / `機種別`（カンマ区切りで複数指定可） |
| D列以降 | 月別PDFハイパーリンク（`updatePdfLinks()` で自動生成） |

### 設定シート
| 設定名 | 内容 |
|---|---|
| PDF保存先フォルダ | Google DriveフォルダURL または Macのローカル絶対パス |
| 対象月 | ドロップダウンで選択（「自動（前月）」= 前月を自動選択） |

## 4. 設定情報の読み込み優先順

### スプレッドシートID
```
環境変数 SPREADSHEET_URL（URLからIDを正規表現で抽出）
  ↓ なければ
環境変数 SPREADSHEET_ID
  ↓ どちらもなければ
エラー終了
```

### PDF保存先
```
設定シートの「PDF保存先フォルダ」
  ├── Google Drive URL → drive_path_map.txt でローカルパスに変換
  └── ローカルパス（/で始まる）→ そのまま使用
  ↓ 設定シートに記載なければ
環境変数 BASE_SAVE_PATH
  ↓ なければ
エラー終了
```

### 対象月
```
環境変数 TARGET_MONTH（YYYYMM形式）
  ↓ なければ
設定シートの「対象月」（「2026年2月」形式 または YYYYMM形式）
  ↓ 「自動（前月）」または未設定なら
前月を自動算出
```

## 5. 処理フロー

```
main()
├── 対象月の決定（TARGET_MONTH → 設定シート → 前月自動）
├── PDF保存先の決定（resolve_save_path）
├── アカウント情報の読み込み（load_accounts）
└── 各アカウントについて download_billing_pdf()
    ├── 既ダウンロード済みチェック → 済みならスキップ
    └── Playwrightブラウザ起動
        ├── セッションファイルがあれば読み込み（/tmp/softbank_session_{電話番号}.json）
        ├── do_login_and_navigate()
        │   ├── BILL_PDF_URLに直アクセス → SoftBankが認証ページにリダイレクト
        │   ├── セキュリティ番号ページ検出時 → _handle_security_code_flow() へ直行
        │   ├── ログインフォームに電話番号・パスワードを入力
        │   ├── _handle_security_code_flow()  ← 1回目認証
        │   ├── 書面発行ページ → PDF印刷ページへ遷移
        │   └── _handle_security_code_flow()  ← 2回目認証（必要な場合）
        ├── セッション保存（/tmp/softbank_session_{電話番号}.json）
        ├── select_target_month()  ← コンボボックスで月を選択
        └── download_pdf_from_page()
            ├── 一括印刷用PDF（doPrintSbmAll）
            ├── 電話番号別PDF（doPrintMsn、idx=2）
            └── 機種別PDF（doPrintMsn、idx=3）
```

## 6. 認証フロー詳細（_handle_security_code_flow）

SoftBankの認証ページ（id.my.softbank.jp）上でループ処理（最大8ステップ）。

```
[ループ開始]
  ├── URLが id.my.softbank.jp 以外 → 認証完了、True を返す
  │
  ├── セキュリティ番号入力欄を検出（input[maxlength="3"]等）
  │   ├── ask_security_code() でコードを取得
  │   ├── 入力欄にコードをfill
  │   ├── input[type="submit"] または button[type="submit"] をクリック
  │   │   ※ <a>タグは対象外（FAQ案内リンクを誤クリック防止）
  │   ├── 待機
  │   └── URLが id.my.softbank.jp 以外 → True を返す
  │
  ├── ラジオボタンを検出（SMS送付先選択）
  │   ├── 電話番号末尾4桁が一致するラジオを選択
  │   └── 一致なし → 先頭を選択
  │
  ├── 「送信する」ボタンをクリック
  │   （セキュリティ番号・送付先・本人確認に関するキーワードがページにある場合）
  │
  └── ログインフォームを検出
      ├── 電話番号・パスワードを再入力
      └── ログインボタンをクリック
```

## 7. セキュリティ番号の入力方法（ask_security_code）

| 環境 | 入力方法 |
|---|---|
| ターミナル直接実行 | `input()` で入力待ち |
| バックグラウンド実行 | `/tmp/softbank_security_code.txt` をポーリング |

ファイル入力の場合：
```bash
echo '854' > /tmp/softbank_security_code.txt
```

- タイムアウト: `SECURITY_CODE_TIMEOUT` 秒（デフォルト300秒）
- 既存ファイルは関数呼び出し時に削除（前回の残りを誤検知しない）

## 8. セッション管理

| 項目 | 内容 |
|---|---|
| 保存場所 | `/tmp/softbank_session_{電話番号}.json`（アカウントごとに独立） |
| 保存タイミング | `do_login_and_navigate()` 成功後 |
| 読み込みタイミング | ブラウザコンテキスト作成時 |
| 有効期限 | SoftBankのセッション期限による（macOS再起動で /tmp が消えてリセット） |

複数アカウントの場合も各自のセッションファイルを使用するため干渉しない。

## 9. ファイル命名規則

```
{YYYYMM}_SoftBank_{電話番号}_{金額}.pdf
```

例：`202602_SoftBank_09048469405_8500円.pdf`

- 請求金額が取得できた場合は金額（例: `8500円`）、できなかった場合は `利用料金明細`
- 電子帳簿保存法の規則（取引年月・取引先・金額を含む）に準拠

## 11. 環境変数

| 変数名 | デフォルト | 説明 |
|---|---|---|
| `SPREADSHEET_URL` | （なし） | スプレッドシートのURL（ブラウザのURLをそのまま貼り付け） |
| `SPREADSHEET_ID` | （なし） | スプレッドシートIDのみ指定する場合（SPREADSHEET_URLが優先） |
| `BASE_SAVE_PATH` | （なし） | PDF保存先（設定シートで管理するため通常不要） |
| `TARGET_MONTH` | 前月自動 | ダウンロード対象月（YYYYMM形式。設定シートより優先） |
| `HEADLESS` | `true` | `false` でブラウザを表示（デバッグ用） |
| `SECURITY_CODE_TIMEOUT` | `300` | SMSコード待機タイムアウト（秒） |

## 13. 既知の制約・注意事項

- **SMS認証は必須**: SoftBankのポリシーでPDFページへのアクセスには毎回SMS認証が必要（セッションが有効な間はスキップ可能）
- **macOS専用**: Googleドライブのローカルパス解決がmacOSの `~/Library/CloudStorage/` 前提
- **ヘッドレスChromium使用**: Playwright標準のChromiumを使用。SoftBankのUA検証により標準ChromeのUser-Agentを偽装している
