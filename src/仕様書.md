# My SoftBank 料金明細PDF自動ダウンロード 仕様書

最終更新: 2026-03-15

---

## 1. 概要

My SoftBankポータルにPlaywrightで自動ログインし、指定月の請求書PDFをダウンロードしてローカルに保存するPythonスクリプト。

### 対応範囲
- My SoftBankの2段階SMS認証（acr_value=1 / acr_value=2）
- スプレッドシートで管理した複数アカウントの一括処理
- 電子帳簿保存法準拠のファイル命名
- セッション保存による実行をまたいだ再認証スキップ

## 3. 設定情報の読み込み優先順

### アカウント情報（電話番号・パスワード・保存先）
```
環境変数 SPREADSHEET_CSV_URL
  ↓ なければ
RTFファイル（MySoftBank_アカウント管理スプシURL.rtf）から URL 抽出
  ↓ どちらもなければ
エラー終了
```

### PDF保存先
```
スプレッドシートの「PDF保存先」列（最初の非空行）
  ├── ローカルパス（/で始まる）→ そのまま使用
  └── Google Drive URL → drive_path_map.txt でローカルパスに変換
  ↓ スプレッドシートに記載なければ
環境変数 BASE_SAVE_PATH
  ↓ なければ
エラー終了
```

## 4. 処理フロー

```
main()
├── 対象月の決定（TARGET_MONTH または前月）
├── PDF保存先の決定（resolve_save_path）
├── アカウント情報の読み込み（load_accounts）
└── 各アカウントについて download_billing_pdf()
    ├── 既ダウンロード済みチェック → 済みならスキップ
    └── Playwrightブラウザ起動
        ├── セッションファイルがあれば読み込み（/tmp/softbank_session.json）
        ├── do_login_and_navigate()
        │   ├── BILL_PDF_URLに直アクセス → SoftBankが認証ページにリダイレクト
        │   ├── ログインフォームに電話番号・パスワードを入力
        │   ├── _handle_security_code_flow()  ← 1回目認証（acr_value=1）
        │   ├── 書面発行ページ → PDF印刷ページへ遷移
        │   └── _handle_security_code_flow()  ← 2回目認証（acr_value=2、必要な場合）
        ├── セッション保存（/tmp/softbank_session.json）
        ├── select_target_month()  ← コンボボックスで月を選択
        └── download_pdf_from_page()
            ├── 一括印刷用PDF（doPrintSbmAll）を優先
            └── なければ電話番号別PDF（doPrintMsn、idx=2）
```

## 5. 認証フロー詳細（_handle_security_code_flow）

SoftBankの認証ページ（id.my.softbank.jp）上でループ処理（最大8ステップ）。

```
[ループ開始]
  ├── URLが id.my.softbank.jp 以外 → 認証完了、True を返す
  │
  ├── セキュリティ番号入力欄を検出（input[maxlength="3"]等）
  │   ├── ask_security_code() でコードを取得
  │   ├── 入力欄にコードをfill
  │   ├── input[type="submit"] または button[type="submit"] をクリック
  │   │   ※ <a>タグは対象外（FAQ案内リンクを誤クリック防止）
  │   ├── 待機
  │   └── URLが id.my.softbank.jp 以外 → True を返す
  │
  ├── ラジオボタンを検出（SMS送付先選択）
  │   ├── 電話番号末尾4桁が一致するラジオを選択
  │   └── 一致なし → 先頭を選択
  │
  ├── 「送信する」ボタンをクリック
  │   （セキュリティ番号・送付先・本人確認に関するキーワードがページにある場合）
  │
  └── ログインフォームを検出
      ├── 電話番号・パスワードを再入力
      └── ログインボタンをクリック
```

## 6. セキュリティ番号の入力方法（ask_security_code）

インタラクティブ環境（stdin が tty）とそれ以外で動作が異なる。

| 環境 | 入力方法 |
|---|---|
| ターミナル直接実行 | `input()` で入力待ち |
| バックグラウンド実行・Claudeから実行 | `/tmp/softbank_security_code.txt` をポーリング |

ファイル入力の場合：
```bash
echo '854' > /tmp/softbank_security_code.txt
```

- タイムアウト: `SECURITY_CODE_TIMEOUT` 秒（デフォルト300秒）
- 既存ファイルは関数呼び出し時に削除（前回の残りを誤検知しない）

## 7. セッション管理

| 項目 | 内容 |
|---|---|
| 保存場所 | `/tmp/softbank_session.json` |
| 保存タイミング | `do_login_and_navigate()` 成功後 |
| 読み込みタイミング | ブラウザコンテキスト作成時 |
| 有効期限 | SoftBankのセッション期限による（macOS再起動で /tmp が消えてリセット） |

セッションが有効な場合、`BILL_PDF_URL` に直接アクセスすると認証なしでPDFページに到達できる。

## 8. ファイル命名規則

```
{YYYYMM}_SoftBank_{電話番号}_{金額}.pdf
```

例：`202602_SoftBank_09048469405_利用料金明細.pdf`

- 請求金額が取得できた場合は金額（例: `8500円`）、できなかった場合は `利用料金明細`
- 電子帳簿保存法の規則（取引年月・取引先・金額を含む）に準拠

## 9. 保存ディレクトリ構成

PDFとデバッグフォルダは同じベースパス（`携帯領収書管理/`）の下に分離して保存される。

```
携帯領収書管理/           ← スプレッドシートの「PDF保存先」で指定するベースパス
├── {YYYY}/
│   └── {MM}/
│       └── {YYYYMM}_SoftBank_{電話番号}_{金額}.pdf
└── debug/               ← エラー時のみ作成（YYYY/MMとは独立）
    └── debug_{YYYYMMDD_HHMMSS}/
        ├── {電話番号}_{YYYYMM}_{エラー種別}.png
        └── {電話番号}_{YYYYMM}_{エラー種別}.txt
```

デバッグの保存先は `save_dir.parent.parent / "debug"` で算出される（`save_dir = base_path/YYYY/MM` の2階層上）。

## 12. 既知の制約・注意事項

- **SMS認証は必須**: SoftBankのポリシーでPDFページへのアクセスには毎回SMS認証が必要（セッションが有効な間はスキップ可能）
- **Google Sheets CSV のキャッシュ**: 「ウェブに公開」CSVは更新後数分経たないと反映されない
- **macOS専用**: Googleドライブのローカルパス解決がmacOSの `~/Library/CloudStorage/` 前提
- **ヘッドレスChromium使用**: Playwright標準のChromiumを使用。SoftBankのUA検証により標準ChromeのUser-Agentを偽装している
