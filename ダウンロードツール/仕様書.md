# 携帯領収書管理 仕様書

最終更新: 2026-03-22（docomo 対応を追加）

---

## 1. 概要

My SoftBank / My Y!mobile / My au / My UQ mobile / My docomo ポータルにPlaywrightで自動ログインし、指定月の請求書PDFをダウンロードしてGoogle Drive APIで直接アップロードするPythonスクリプト。

### 対応範囲
- My SoftBank / My Y!mobile の2段階SMS認証（acr_value=1 / acr_value=2）
- My au / My UQ mobile のau ID認証（2段階認証: 確認コードまたはワンタイムURL）
- My docomo のdアカウント認証（2段階認証: SMS/メールで6桁セキュリティコード）
- Googleスプレッドシート（サービスアカウント認証またはOAuth2認証）で管理した複数回線の一括処理
- 電子帳簿保存法準拠のファイル命名
- 電話番号ごとに独立したセッション保存による再認証スキップ
- Google Drive APIによるPDF直接アップロード（OAuth2優先、サービスアカウントフォールバック）
- 各種設定の自動化（`.env` 自動生成、機密ファイル自動検索・ダウンロード）

### キャリア間の差分

| 項目 | SoftBank | Y!mobile | au | UQ mobile | docomo |
|---|---|---|---|---|---|
| carrier_family | `softbank` | `softbank` | `au` | `au` | `docomo` |

## 3. スプレッドシート構成

2つのスプレッドシートを使用する:
- **認証情報管理シート**（GASを配置する側）: 設定・認証情報・PDFリンク管理
- **回線管理スプレッドシート**（別スプレッドシート）: 月別の電話番号・解約済・運用端末・名義等

スプレッドシートへのアクセスはGCPサービスアカウント（`service_account.json`）またはOAuth2（`client_secrets.json`）で認証。
URLは `ダウンロードツール/.env` の `SPREADSHEET_URL` に認証情報管理シートのURLを設定する。

### 認証情報シート（キャリア共通）
| 列 | 内容 |
|---|---|
| 電話番号 | SoftBank / Y!mobile / au / UQ mobile / docomo の電話番号（ハイフンなし） |
| キャリア | `SoftBank` / `Ymobile` / `au` / `UQmobile` / `docomo`（サイドバーから自動設定） |
| PDFの種類 | SB/YM: `電話番号別` / `一括` / `機種別`、au/UQ: `請求書` / `領収書` / `支払証明書`、docomo: `適格請求書` / `利用内訳`（カンマ区切りで複数指定可） |
| 運用端末 | SMS認証時に表示する端末名（サイドバー保存時に回線管理スプレッドシートから自動設定） |
| 状態 | 有効回線は「契約中」、解約済回線は「解約済」と表示・グレーアウト |
| ログインID | 回線管理スプレッドシートの「ID」列から自動設定。SB/YM: SoftBank ID、au/UQ: au ID、docomo: dアカウントID |

パスワードは設定シートで一元管理（認証情報シートにはパスワード列なし）。
サイドバーで電話番号を選択・保存すると認証情報シートに書き込まれる。

**ログインID（SoftBank ID / au ID / dアカウントID）について:**
- au/UQ回線: au IDでログインする（電話番号ではログインできないため）
- SoftBank/Ymobile回線: ログインIDがあればSoftBank IDでログイン、なければ電話番号でログイン
- 解約済のSoftBank/Ymobile回線: 電話番号ではログインできないためSoftBank IDが必須（未設定時はエラー）
- docomo回線: dアカウントIDでログイン（ログインIDがなければ電話番号を使用）

**解約済回線の扱い:**
- サイドバーで解約済回線も表示・選択可能（「（解約済）」表示付き）
- 選択された解約済回線はPDFの種類付きで認証情報シートに書き込まれ、ダウンロード対象になる
- 認証情報シート上では解約済回線はグレーアウト＋取り消し線で表示される

### 設定シート（キャリア共通）
| 設定名 | 内容 |
|---|---|
| このスプレッドシートURL | Pythonの SPREADSHEET_URL に使用（自動設定） |
| 回線管理スプレッドシート | 回線管理表のURL（別スプレッドシート） |
| PDF保存先フォルダ | Google DriveフォルダURL（推奨）または ローカル絶対パス |
| パスワード | SoftBank / Y!mobile 共通のログインパスワード |
| au/UQパスワード | au / UQ mobile 共通のau IDパスワード |
| au暗証番号 | au / UQ mobile の4桁暗証番号（請求書閲覧時に必要な場合あり） |
| dアカウントパスワード | docomo のdアカウントパスワード |
| 対象月 | ドロップダウンで選択（「自動（前月）」= 前月を自動選択） |

### ダウンロード履歴シート
| 列 | 内容 |
|---|---|
| 日時 | ダウンロード実行日時（`YYYY-MM-DD HH:MM`） |
| キャリア | `SoftBank` / `Ymobile` / `au` / `UQmobile` / `docomo` |
| 電話番号 | 対象の電話番号 |
| 対象月 | `YYYYMM` 形式 |
| ファイル名 | ダウンロード済みPDFの最終ファイル名（金額取得・リネーム後の名前） |
| 結果 | `成功` / `失敗` |

### SoftBankリンク / Ymobileリンク / auリンク / UQmobileリンク / docomoリンク シート
| 列 | 内容 |
|---|---|
| 電話番号 | 電話番号（サイドバー保存時に認証情報から自動同期） |
| 名義 | 契約者名（回線管理表から自動設定） |

### 回線管理スプレッドシート（別スプレッドシート）
月別シート（「3月」等）にキャリアごとのセクション分け構造で電話番号を管理。

| 列 | 内容 |
|---|---|
| 電話番号 | 電話番号（ハイフンあり/なしどちらでもOK） |
| 解約済 | `TRUE` で解約済として処理対象から除外される |
| 運用端末 | SMS認証時に表示する端末名 |
| 名義 | 契約者名（リンクシートに自動反映） |

- キャリアごとにセクション分け（「ワイモバイル」「ソフトバンク」ラベル行 → ヘッダー行 → データ行）
- 解約済の行はサイドバー保存時に認証情報シートから自動除外される

## 4. Drive API認証の仕組み

### 認証優先順

```
client_secrets.json が存在する
  ↓ Yes
OAuth2認証（google-auth-oauthlib）
  ├── drive_oauth_token.json が存在する → トークンを再利用
  └── 存在しない → ブラウザでGoogleアカウント認証 → トークンを保存
  ↓ No
サービスアカウント認証（service_account.json）
```

### OAuth2認証の特徴
- ユーザー自身のGoogleアカウントのDriveに直接アップロードされる（プライバシー保護）
- Driveフォルダへの共有設定が不要
- `storageQuotaExceeded` エラーが発生しない

### サービスアカウント認証の特徴
- `storageQuotaExceeded` エラーが発生した場合、プロジェクトフォルダにローカル保存してGoogle Drive同期にフォールバック
- Driveフォルダをサービスアカウントの `client_email` に「編集者」として共有する必要がある

### 機密ファイルの自動検索

スクリプト起動時に `service_account.json` と `client_secrets.json` を `ダウンロードツール/` フォルダから検索する。

## 5. 共通ユーティリティ・自動化機能

### `shared_utils.py`（全ロジック集約モジュール）

`ダウンロードツール/` に配置。すべてのダウンロードロジックを集約し、`download.py` は `CarrierConfig` を定義して `run_main()` を呼ぶだけの薄いラッパーとして動作する。

#### 主要クラス

| クラス | 役割 |
|---|---|
| `CarrierConfig` | キャリア固有の定数（URL・ドメイン・ファイルプレフィックス・carrier_family等）を保持するfrozen dataclass |
| `BillingContext` | 実行時の状態（設定値・DriveContext・端末マップ等）を保持するdataclass |
| `DriveContext` | Google Drive APIのフォルダ操作・アップロード・存在確認 |

#### 主要関数

| 関数 | 機能 |
|---|---|
| `create_billing_context()` | CarrierConfig + script_dir からBillingContextを初期化 |
| `run_main()` | 全回線のPDFダウンロードを実行するメインエントリポイント |
| `do_login_and_navigate()` | ログイン → 2FA → PDFページ遷移 |
| `download_billing_pdf()` | 1回線分のPDFダウンロード処理 |
| `retry_with_backoff()` | 指数バックオフ付きリトライ |
| `strip_hyphens()` | 電話番号のハイフン除去・先頭0補完 |
| `parse_pdf_types()` | PDFの種類列のパース |
| `write_download_history()` | ダウンロード結果を履歴シートに記録 |

### `.env` 自動生成（`bootstrap_env_from_gsheet()`）

`.env` ファイルが存在しない場合に自動生成する。

1. スクリプトフォルダ内の `.gsheet` ファイルを検索
2. `.gsheet` ファイルのJSONから `doc_id` を抽出
3. `SPREADSHEET_URL={url}` を `.env` に書き込む

### `_bootstrap_service_account()`

`service_account.json` が見つからない場合に兄弟フォルダを自動検索する。

### `_bootstrap_client_secrets()`

`client_secrets.json` が見つからない場合に以下を試みる：

1. 兄弟フォルダを自動検索
2. PDF保存先フォルダ（Driveフォルダ）に `client_secrets.json` が存在すれば自動ダウンロードしてスクリプトフォルダに配置

### `_bootstrap_spreadsheet_permission()`

スプレッドシートへのアクセスで403エラー（権限不足）が発生した場合：

1. 共有に必要なメールアドレスをクリップボードに自動コピー
2. スプレッドシートのURLをブラウザで自動起動
3. 共有設定を手動で追加するよう案内する

## 5-1. HTMLサイドバーによる電話番号管理

メニュー「携帯領収書管理 ツール」→「ダウンロード対象の電話番号を管理」でHTMLサイドバーを開き、回線管理スプレッドシートの月別シートから電話番号を読み込んで選択する。

### データフロー

```
[回線管理スプレッドシート（別スプシ）]
    月別シート → 電話番号・解約済・キャリア・運用端末・名義・ID
         ↓ HTMLサイドバーで対象月のシートを読み込み・選択・保存
[認証情報管理シート]
    認証情報シート: 電話番号・キャリア・PDFの種類・運用端末・状態
    リンクシート:   電話番号・名義・月別PDFリンク
    設定シート:     パスワード（共通）
         ↓ Python が読む
[Pythonスクリプト]
```

### サイドバーの「保存」で行われること

1. **認証情報シート**: 全データをクリアして再書き込み
   - 選択中の契約中回線 → 通常表示
   - 選択中の解約済回線 → PDFの種類付きでグレー+取り消し線表示
   - 未選択の解約済回線 → PDFの種類なしでグレー+取り消し線表示
2. **リンクシート**: 新しい電話番号と名義を追加（既存行やPDFリンクは削除しない）。解約済回線のグレーアウトは追加のみ（対象月を切り替えても過去のグレーアウトは消えない）

### 解約済回線の処理

- サイドバーには解約済回線も表示（「（解約済）」ラベル＋グレー表示）、選択・PDFの種類指定可能
- 設定シートの対象月に対応する月シートの回線のみ表示
- 初期状態は全番号チェック済み（個別に外してイレギュラー対応可能）
- 認証情報シート: 解約済回線はグレーアウト＋取り消し線で表示
- リンクシート: 解約済回線のA,B列をグレーアウト＋取り消し線で表示（追加のみ、解除しない）
- Pythonは解約済回線もダウンロード対象に含める（SoftBank IDまたはau IDでログイン）

### セットアップ手順

1. Apps Script に `setup_spreadsheet.gs` を貼り付けて保存
2. `setupSheet()` を実行
3. 設定シートに「回線管理スプレッドシート」URLと「パスワード」を入力
4. メニュー「携帯領収書管理 ツール」→「ダウンロード対象の電話番号を管理」で対象を選択・保存

### 端末情報のキャッシュ

認証情報シートの「運用端末」列（サイドバー保存時に回線管理表から自動設定）を `_phone_device_map`（電話番号 → 運用端末名）としてキャッシュする。SMS認証時の端末表示に利用される。

## 8. 認証フロー詳細（_handle_security_code_flow）

各キャリアの認証ページ（SoftBank: `id.my.softbank.jp` / Y!mobile: `id.my.ymobile.jp`）上でループ処理（最大8ステップ）。

```
[ループ開始]
  ├── URLが認証ドメイン（auth_domain）以外 → 認証完了、True を返す
  │
  ├── セキュリティ番号入力欄を検出（input[maxlength="3"]等）
  │   ├── ask_security_code() でコードを取得
  │   ├── 入力欄にコードをfill
  │   ├── input[type="submit"] または button[type="submit"] をクリック
  │   │   ※ <a>タグは対象外（FAQ案内リンクを誤クリック防止）
  │   ├── 待機
  │   └── URLが認証ドメイン以外 → True を返す
  │
  ├── ラジオボタンを検出（SMS送付先選択）
  │   ├── 電話番号末尾4桁が一致するラジオを選択
  │   └── 一致なし → 先頭を選択
  │
  ├── 「送信する」ボタンをクリック
  │   （セキュリティ番号・送付先・本人確認に関するキーワードがページにある場合）
  │
  └── ログインフォームを検出
      ├── 電話番号・パスワードを再入力
      └── ログインボタンをクリック
```

## 9. セキュリティ番号の入力方法（ask_security_code）

SMS認証が必要な場合、ターミナルに電話番号と端末名を表示して入力を促す。端末名は `_phone_device_map` から取得する。

```
============================================================
  📱 SMS認証が必要です
  電話番号: 09047695015
  端末    : iPhoneAir
  SMSに届いた3桁のセキュリティ番号を入力してください
  ターミナル入力 または 以下のコマンドで渡してください:
    echo '123' > /tmp/softbank_security_code.txt   ← SoftBankの場合
    echo '123' > /tmp/ymobile_security_code.txt    ← Y!mobileの場合
============================================================
```

| 環境 | 入力方法 |
|---|---|
| ターミナル直接実行 | `input()` で入力待ち |
| バックグラウンド実行 | `{tempdir}/{キャリア}_security_code.txt` をポーリング（`softbank_` / `ymobile_`） |

ファイル入力の場合：
```bash
echo '854' > /tmp/softbank_security_code.txt   # SoftBankの場合
echo '854' > /tmp/ymobile_security_code.txt    # Y!mobileの場合
```

- タイムアウト: `SECURITY_CODE_TIMEOUT` 秒（デフォルト60秒）
- 既存ファイルは関数呼び出し時に削除（前回の残りを誤検知しない）
- `_phone_device_map` に端末名がなければ電話番号のみ表示される

## 11. ファイル命名規則

```
{YYYYMM}_{会社名}_{電話番号}_{金額}円(税抜).pdf
```

例：`202601_ソフトバンク株式会社_09014704347_1880円(税抜).pdf`

- 金額はPyMuPDFでPDFテキストを直接抽出して取得（OCR不要）
- ダウンロード時に自動取得、または `python3 update_amounts.py` で後から更新可能
- 取得できなかった場合は `_利用料金明細.pdf` のまま残る
- 金額はPDFに記載されている値をそのまま使用（余計な計算はしない）
  - SoftBank/Ymobile: PDFの「計」（個別回線の小計）
  - au/UQ: PDFの「10%消費税の課税対象額」（個別回線の課税対象額）
- GASのOCR金額取得メニューは廃止済み
- 電子帳簿保存法の規則（取引年月・取引先・金額を含む）に準拠

## 15. 既知の制約・注意事項

- **SMS認証は必須**: SoftBankのポリシーでPDFページへのアクセスには毎回SMS認証が必要（セッションが有効な間はスキップ可能）
- **OAuth2認証推奨**: サービスアカウント認証では `storageQuotaExceeded` が発生する場合がある。OAuth2認証ではユーザー自身のDriveにアップロードするため発生しない
- **ヘッドレスChromium使用**: Playwright標準のChromiumを使用。SoftBankのUA検証により標準ChromeのUser-Agentを偽装している
- **機密ファイルは `ダウンロードツール/` に配置**: `service_account.json` / `client_secrets.json` / `drive_oauth_token.json` はすべて `ダウンロードツール/` フォルダに置く

## 16. 今後の対応予定

- **機種別PDFのダウンロード対応**: SoftBank/Ymobileの「機種別」PDFは現在特殊な扱いで、ダウンロード処理は電話番号別と同じ `doPrintMsn` リンクから取得するが、請求書としての本格的な対応は未実装。今後、機種別の請求書ダウンロードとリンクシートへの反映を実装予定。現在リンクシートでは `_機種別.pdf` サフィックスを除外している（`_updatePdfLinks_` の `excludeSuffixes`）
